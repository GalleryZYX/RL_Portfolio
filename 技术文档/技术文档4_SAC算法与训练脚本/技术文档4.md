## 1. 背景

在完成 PPO 算法验证（技术文件2、3）后，项目新增了基于 **SAC（Soft Actor-Critic）** 的训练方案。SAC 是一种 off-policy 最大熵深度强化学习算法，由本文件两位作者负责实现和调优。本章介绍 SAC 算法原理、`train_sac_portfolio.py` 的设计架构，以及预期运行结果。

## 2. SAC 算法简介

### 2.1 核心思想

SAC 在标准 Actor-Critic 框架上引入**最大熵目标**，在最大化累计奖励的同时最大化策略的熵：

```
J(π) = Σ_t E[(r_t + α · H(π(·|s_t))]
```

其中 `H(π)` 为策略的熵，`α` 为熵温度系数（ent_coef），控制探索与利用的平衡。

### 2.2 为何选 SAC

| 特性 | SAC | PPO |
|------|-----|-----|
| 学习方式 | **Off-policy**（经验回放） | On-policy（在线采样） |
| 样本效率 | 高 — 每条数据可多次重用 | 低 — 每次更新后需重新采样 |
| 探索机制 | 最大熵 — 天然鼓励探索 | 策略熵正则项 — 需手动调参 |
| 超参数敏感度 | 较高，需仔细调 α | 较低，默认参数通常可用 |
| 适合场景 | 数据量有限、高维连续空间 | 稳定性要求高、可大量在线采样 |

本项目训练数据仅 717 个交易日，SAC 的 off-policy 回放机制能更高效利用有限数据。此外，最大熵目标天然鼓励持仓多样化，不易过早坍缩为单一股票。

### 2.3 网络结构

SB3 的 SAC 使用三个网络：

- **Actor（策略网络）**：输出高斯分布的均值和对数标准差，经 tanh 压缩到 `[-1, 1]`，匹配环境的动作空间
- **Twin Q-Networks（双 Critic）**：两套独立的 Q 网络，取较小值抑制高估偏差
- **Target Q-Networks**：软更新的目标网络，稳定训练

默认隐藏层结构为 `[256, 256]`（Actor 和 Critic 共用），适配 612 维输入。

## 3. 代码设计

### 3.1 文件位置

```
scripts/train_sac_portfolio.py    # SAC 训练主脚本
```

### 3.2 架构流程图

```
命令行参数解析 (parse_args)
    │
├─ 数据加载：pd.read_csv(train_csv)
│      │  51 只股票，717 个交易日
│      │
├─ 训练环境 → VecNormalize(training=True)
│      │  PortfolioEnv + Monitor + DummyVecEnv
│      │  观测/奖励归一化（running stats 持续更新）
│      │
├─ 评估环境 → VecNormalize(training=True)    ← 独立实例！
│      │  同一份训练数据，独立的 DummyVecEnv
│      │  避免 EvalCallback 污染训练的归一化统计量
│      │
├─ SAC 模型（MlpPolicy，双 Q 网络，256×256）
│      │
├─ EvalCallback（每 10K 步评估 3 个 episode）
│      │  使用独立 eval 环境，不影响训练
│      │  自动保存最佳模型到 models/sac_best/
│      │
├─ model.learn(total_timesteps=100K)
│      │
├─ 保存模型 + VecNormalize 统计量
│      │  models/sac_portfolio.zip
│      │  models/sac_portfolio_vecnorm.pkl
│      │
├─ 样本内评估（训练数据，3 episodes）
│      │  VecNormalize.training = False（冻结统计量）
│      │  输出：总收益、夏普比率、最大回撤、胜率
│      │
└─ 样本外评估（测试数据，2026 年）
       VecNormalize.training = False
       输出：样本外指标
```

### 3.3 关键设计决策

#### 3.3.1 训练/评估环境隔离

训练和 EvalCallback 使用**独立的 VecNormalize 实例**，避免评估过程污染训练归一化统计量：

- **训练环境**（`train_vec_norm`）：running stats 持续更新，用于采样和 replay buffer 存储
- **评估环境**（`eval_vec_norm`）：独立维护统计量，EvalCallback 在此运行，互不干扰

#### 3.3.2 最终评估时冻结 VecNormalize

`evaluate_portfolio()` 在评估前设置 `vec_norm.training = False`，评估后恢复。确保多 episode 评估中归一化统计量不会漂移。

#### 3.3.3 Off-policy 经验回放

SAC 使用 `buffer_size=100,000` 的回放缓冲区。由于每个 episode 仅 713 步，缓冲区足以存储全部训练经验。`learning_starts=2000` 确保在开始学习前有约 3 个 episode 的随机探索数据。

## 4. 使用方法

### 4.1 基本命令

```bash
# 激活环境
conda activate rl_env

# 默认参数运行（100K 步，GPU）
python scripts/train_sac_portfolio.py

# CPU 运行
python scripts/train_sac_portfolio.py --device cpu

# 增加训练步数
python scripts/train_sac_portfolio.py --total-timesteps 500_000

# 查看全部可调参数
python scripts/train_sac_portfolio.py --help
```

### 4.2 超参数一览

#### 环境参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--train-data` | `data/rl_train_data_2023_2025.csv` | 训练数据路径 |
| `--test-data` | `data/rl_test_data_2026_now.csv` | 测试数据路径 |
| `--initial-cash` | `1,000,000.0` | 初始资金 |
| `--commission` | `0.001` | 单边交易成本（0.1%） |
| `--window` | `20` | z-score 滚动窗口（交易日） |

#### SAC 核心超参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--total-timesteps` | `100,000` | 训练总步数 |
| `--learning-rate` | `3e-4` | Actor/Critic/Alpha 学习率 |
| `--buffer-size` | `100,000` | Replay buffer 容量 |
| `--batch-size` | `256` | 每次梯度的 mini-batch 大小 |
| `--gamma` | `0.99` | 折扣因子 |
| `--tau` | `0.005` | 目标网络软更新系数 |
| `--ent-coef` | `auto` | 熵温度系数（auto = 自适应调节） |
| `--learning-starts` | `2000` | 开始学习前收集的步数 |
| `--train-freq` | `1` | 每 N 步训练一次 |
| `--gradient-steps` | `1` | 每次训练的梯度步数 |
| `--net-arch` | `256,256` | 隐藏层结构（逗号分隔） |

#### 训练流程参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--seed` | `42` | 随机种子 |
| `--device` | `cuda:0` | 训练设备 |
| `--eval-freq` | `10,000` | 每 N 步评估一次 |
| `--eval-episodes` | `3` | 每次评估的 episode 数 |
| `--models-dir` | `models` | 模型保存目录 |
| `--log-dir` | `runs` | TensorBoard 日志目录 |

**关键观察：**

1. **稳定收敛**：奖励从负值单调递增至 +5.185，每次评估的 3 个 episode 结果几乎一致（标准差 < 0.001），说明策略确定性高

3. **未达平台期**：100K 步时奖励仍在上升，说明更多训练（300K~500K 步）可能进一步提升性能

## 7. 与 PPO 方案的对比

| 维度 | SAC（本文档） | PPO（技术文件2） |
|------|-------------|----------------|
| 算法类型 | Off-policy | On-policy |
| 经验利用 | Replay buffer 重复使用 | 每次采样后丢弃 |
| 探索机制 | 最大熵（自动平衡） | 熵正则项（手动调参） |
| 默认训练步数 | 100K（约 140 episodes） | 1M（约 1400 episodes） |
| 内存占用 | 需 replay buffer（~100K transitions） | 仅保留当前 rollout |
| 网络结构 | `[256, 256]`（Actor + 双 Critic） | `{"pi": [256,256], "vf": [256,256]}` |
| Eval 环境 | 独立 VecNormalize 实例 | 独立 VecNormalize 实例 |
| 保存文件 | `sac_portfolio.zip` + `vecnorm.pkl` | `ppo_portfolio.zip` + `vecnorm.pkl` |

两种算法各有优势：PPO 更稳定、超参数鲁棒；SAC 样本效率更高、探索更充分。建议在项目最终阶段对比两者的样本外表现，选择更优方案。

## 8. 注意事项

1. **CUDA 支持**：脚本默认使用 `cuda:0`，若无 GPU 会自动回退 CPU，也可手动指定 `--device cpu`
2. **网络格式**：SAC 的 `--net-arch` 参数接受逗号分隔的整数列表（如 `256,256`），不同于 PPO 的字典格式
3. **VecNormalize 隔离**：训练与评估环境使用独立的 VecNormalize 实例，最终评估时冻结统计量
4. **数据文件**：`data/*.csv` 未被 Git 跟踪，需确保本地数据文件存在
5. **可复现性**：固定 `--seed 42` 确保相同数据和超参数下结果可复现
