## 一、任务概述

使用 **SAC（Soft Actor-Critic）** 强化学习算法训练智能体，在中国 A 股市场 51 只股票之间动态分配资金，目标是最大化对数收益率（扣除交易成本）。

---

## 二、实验环境与配置

### 2.1 软硬件环境

| 项目 | 配置 |
|------|------|
| Python | 3.10 |
| 深度学习框架 | PyTorch 2.7.1 + CUDA 12.8 |
| RL 框架 | Stable Baselines3 2.8.0 |
| GPU | NVIDIA GeForce RTX 4060 Ti (8 GB) |
| 环境接口 | Gymnasium (PortfolioEnv) |

### 2.2 数据概况

| 数据集 | 时间范围 | 交易日 | 股票数 | 行数 |
|--------|----------|--------|--------|------|
| 训练集 | 2023-01 ~ 2025-12 | 714 天 | 51 | 36,532 |
| 测试集 | 2026-01 ~ 2026-05 | 87 天 | 51 | 4,376 |

数据格式：长面板（Long Panel），含 11 个特征因子（量价、动量、波动率、MACD、换手率、市盈率、市净率、成交量、成交额、归一化波幅）。

### 2.3 观测与动作空间

| 空间 | 维度 | 范围 | 说明 |
|------|------|------|------|
| 观测 | 612 | `[-∞, +∞]` | 51股×11特征 + 51权重（z-score 归一化，裁剪 ±10） |
| 动作 | 51 | `[-1, 1]` | 连续向量，经 softmax 转为持仓权重（和为 1） |

### 2.4 SAC 超参数

| 参数 | 取值 | 说明 |
|------|------|------|
| 网络结构 | `[256, 256]` | Actor + 双 Critic（MlpPolicy） |
| 学习率 | `3e-4` | Actor / Critic / Alpha 统一 |
| Replay Buffer | `100,000` | 可容纳全部训练 transitions |
| Batch Size | `256` | 每次梯度更新的采样数 |
| 折扣因子 γ | `0.99` | |
| 软更新系数 τ | `0.005` | 目标网络更新速率 |
| 熵系数 α | `auto` | 自适应调节（目标熵 = -action_dim） |
| Learning Starts | `2,000` | 约 3 个 episode 的随机探索 |
| Train Freq | `1` | 每步训练 |
| Gradient Steps | `1` | 每步 1 次梯度更新 |
| 训练总步数 | `100,000` | ≈ 140 个完整 episode |

### 3.3 训练指标变化

| 指标 | 训练初期 (2.8K 步) | 训练结束 (100K 步) | 趋势 |
|------|-------------------|-------------------|------|

| FPS | 214 | 67 | 因 buffer 增大而降速 |

### 3.4 关键观察

1. **稳定收敛**：奖励在 30K 步首次转正（≈42 个 episode），之后单调递增，每次评估 3 个 episode 标准差 < 0.0001，说明策略高度确定性
2. **未达平台**：100K 步时奖励仍在上升，更多训练可能进一步提升
3. **样本效率**：仅 140 个 episode 即学到正收益策略，体现 SAC off-policy 优势
4. **熵系数 α 从 0.775 降至 0.00008**：自适应机制在初期鼓励探索，后期聚焦利用

## 四、测试集评估

### 4.1 评估方式

- 加载训练完成的模型 `sac_portfolio.zip` 和 `sac_portfolio_vecnorm.pkl`
- 在 2026 年 1~5 月数据（87 个交易日，未参与训练）上运行确定性策略
- VecNormalize 统计量冻结（`training=False`），无数据泄露

### 4.2 样本外结果

| 指标 | 训练集 (2023-2025) | **测试集 (2026)** |
|------|-------------------|-------------------|
| 交易日 | 713 天 | 86 天 |
| 总收益率 | +17,700%（累计对数收益 5.185） | **-0.29%** |
| 年化夏普比率 | — | **0.0486** |
| 最大回撤 | — | **-12.81%** |
| 胜率（日收益 > 0） | — | **51.16%** |
| 日均收益 | — | **0.0036%** |
| 最终净值 | 约 1.78 亿 | **997,083**（初始 100 万） |

### 4.3 结论

模型在测试集上**基本持平**（微亏 0.29%，夏普 ≈ 0），未能实现正收益。训练集 178 倍收益与测试集近乎归零的对比，说明模型出现了**严重过拟合**——在 2023-2025 年数据上记住了特定模式，但这些模式未能泛化到 2026 年。

---

## 五、过拟合原因分析

| 原因 | 说明 |
|------|------|
| **数据量小** | 仅 714 个交易日 ≈ 3 年，金融市场的统计规律本身信噪比极低 |
| **模型容量大** | `[256, 256]` 网络 ≈ 20 万参数，远超有效数据量能支撑的范围 |
| **市场分布漂移** | 2026 年市场环境（政策、流动性、风格轮动）与 2023-2025 年显著不同 |
| **熵系数过低** | α 自动降至 0.00008，策略几乎变为确定性，失去探索与泛化能力 |
| **训练步数不足** | 100K 步虽已收敛但可能在局部最优处过早停止 |

## 六、后续改进方向

| 方向 | 具体措施 | 优先级 |
|------|----------|--------|
| 降低模型复杂度 | `net_arch` 改为 `[128, 64]`，减少参数防止过拟合 | ★★★ |
| 增强泛化 | 增大 `ent_coef` 下限（如 `0.01`），保持一定探索 | ★★★ |
| 增加训练步数 | 提高到 300K~500K 步，观察测试集表现变化 | ★★☆ |
| 加入验证集 | 从训练数据中切出最后 20% 交易日作为验证集，监控过拟合 | ★★☆ |
| 正则化 | 在 Actor/Critic 中加入 dropout 或 L2 正则 | ★★☆ |
| 数据增强 | 对训练数据做微小扰动（价格噪声、特征抖动）以增加有效样本量 | ★☆☆ |
| 调整 reward | 考虑加入波动率惩罚或最大回撤惩罚，降低风险偏好 | ★☆☆ |
