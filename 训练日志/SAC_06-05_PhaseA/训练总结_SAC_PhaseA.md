# SAC Phase A 训练总结：抗过拟合改进

**日期：** 2026-06-05  
**编写人：** 胡锦宏、刘骏哲元  
**关联训练：** `python scripts/train_sac_portfolio.py`（300K 步）

---

## 一、背景

初次 SAC 训练（06-04）发现严重过拟合：训练集 178 倍收益 vs 测试集 -0.29%。Phase A 目标：在不引入新数据的前提下，通过缩小网络、正则化、验证集早停等手段消除过拟合。

---

| 文件 | 改动项 | 说明 |

### 2.2 详细变更

| # | 参数/逻辑 | Phase 0（旧） | Phase A（新） | 理由 |
|---|-----------|:--------:|:--------:|------|
| A1 | net_arch | `[256,256]` | `[64,32]` | 70万→4万参数，防止记忆 |
| A2 | ent_coef | `auto`（→0.00008） | `0.05`（固定） | 保持策略熵，防止坍缩 |
| A3 | weight_decay | 无 | `1e-4` | L2 正则化 |
| A4 | max_grad_norm | 无 | 移除（SB3 SAC 不支持） | — |
| A5 | 验证/早停 | EvalCallback（训练集） | ValidationCallback（2025验证集）+ patience=10 | 检测过拟合，选择最佳模型 |
| A6 | 特征 | 11个（含 vol, amount） | 12个（去掉 vol/amount，加 gap/intraday_amp/close_loc） | 去绝对量纲噪声，加尺度不变特征 |
| A7 | OOS 评估 | 1 episode | 5 episodes + Q1/Q2 子期间 | 统计稳健性 |
| A8 | 超参数 | lr=3e-4, batch=256, 100K步 | lr=1e-3, batch=128, 300K步 | 适配小网络 |

## 五、发现与诊断

### 5.1 成功点

1. **过拟合已消除**：训练和测试收益从"天上地下"变为水平接近，核心目标达成
2. **测试指标全面改善**：收益转正、夏普提升、回撤降低
3. **验证早停机制正确工作**：最佳模型 @ 80K 步被正确识别和保存
4. **Q2 表现优异**：+7.39% 收益、夏普 3.20，说明模型在某些市场环境下能找到有效信号

### 5.2 新问题：欠拟合

| 症状 | 原因 |
|------|------|
| 训练收益为负（-2.37%） | `[64,32]` 网络过小，663 维→64→32→51 的信息瓶颈太紧 |
| 验证奖励完全持平 0.24 | 模型容量不足以进一步优化 |
| 训练 episode 奖励 -0.32 | 无法在训练数据上取得正收益 |

### 5.3 Bug

- **早停未实际停止训练**：`model.stop_training = True` 在 SAC off-policy 学习循环中未被及时检查，导致训练持续至 300K 步。不影响结果（最佳模型已在 80K 步保存）。

## 六、下一步建议

Phase A 消除了过拟合但矫枉过正。Phase A v2 建议折中参数：

```bash
python scripts/train_sac_portfolio.py \
  --net-arch 128,64 \
  --ent-coef 0.01 \
  --learning-rate 3e-4 \
  --total-timesteps 300_000
```

| 参数 | Phase A | Phase A v2 建议 | 理由 |
|------|:------:|:------:|------|
| net_arch | [64,32] | **[128,64]** | 适度恢复容量（~10万参数） |
| ent_coef | 0.05 | **0.01** | 降低探索，允许收敛 |
| learning_rate | 1e-3 | **3e-4** | 恢复默认，更稳定 |
| total_timesteps | 300K | **300K** | 保持 |
