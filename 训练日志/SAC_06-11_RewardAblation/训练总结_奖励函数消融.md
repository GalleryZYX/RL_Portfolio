# 奖励函数消融实验总结

**日期:** 2026-06-11  
**阶段:** Phase E — 奖励函数改造（移植 PPO 版奖励分量）  
**目的:** 对比 PPO 组的奖励函数，引入波动率惩罚、夏普奖励、换手率惩罚，检验是否能突破 SAC ≈ 等权再平衡的瓶颈

## 一、修改内容

### 对比 PPO 环境 `portfolio_env_ppo.py`

PPO 团队的奖励函数比 SAC 多了四个分量：

| 分量 | SAC 原有 | PPO 版 | 作用 |
|------|:--:|:--:|------|
| 对数收益 `log(1+r)` | ✓ | ✓ | 基础回报 |
| 波动率惩罚 | ✗ | `volatility_penalty × rolling_vol` | 惩罚高波动 |
| 夏普奖励 | ✗ | `sharpe_bonus × rolling_sharpe` | 奖励风险调整收益 |
| 换手率惩罚 | ✗ | `turnover_penalty × turnover` | 惩罚频繁交易 |
| 回撤惩罚 | ✓ | ✓ | 惩罚回撤 |
| 集中度惩罚 | ✓ | ✗ | SAC 独有，鼓励分散 |

### 移植内容

1. **`envs/portfolio_env.py`**：新增 5 个参数 (`volatility_penalty`, `sharpe_bonus`, `turnover_penalty`, `risk_free_rate`, `risk_window`)，移植 `_compute_risk_metrics()` 方法
2. **`scripts/train_sac_portfolio.py`**：新增对应 CLI 参数
3. **`scripts/walk_forward.py`**：传递新参数到环境

## 四、核心发现

### 1. 关系非单调
不是夏普奖励越大越好。sb=0.005 最优，sb=0.01 反而下降，说明存在最优区间。

### 2. 波动惩罚是双刃剑
Sweep C（vol=0.02）收益 +0.42%，甚至低于 Baseline（+0.47%）。过度惩罚波动让模型畏缩不敢持仓，错失上涨收益。

### 3. 最佳改善仅 +0.2 个百分点
从 +0.47% → +0.67%，夏普 0.17 → 0.20。方向上正面，但量级远不足以突破 SAC ≈ EW 的格局。

### 4. 瓶颈在特征，不在奖励
相比特征工程阶段（11→17 特征使 WF 均值从 3.18% → 3.54%），奖励函数改造的改善更小。输入信息的限制才是根本瓶颈。

---

## 五、结论

- **最佳参数组合**：`volatility_penalty=0.01, sharpe_bonus=0.005, turnover_penalty=0.001`，已写入训练脚本默认值
- **边际改善可保留**：+0.2% 也是改善，作为默认配置使用
- **要突破 SAC ≈ EW**：必须从输入特征（更长历史 / 外部数据）或模型架构（LSTM / Transformer）层面着手

### 奖励公式（最终版）

```python
reward = (
    log_return                              # 基础对数收益
    - volatility_penalty * rolling_vol      # 惩罚高波动
    - turnover_penalty * turnover           # 惩罚频繁换手
    + sharpe_bonus * rolling_sharpe         # 奖励风险调整收益
    - drawdown_penalty * drawdown           # 惩罚回撤
    - concentration_penalty * HHI           # 惩罚集中持仓
)
```

### 风险指标计算

- `rolling_volatility`：近 20 日超额收益年化波动率（×√252）
- `rolling_sharpe`：近 20 日年化夏普比率（clip [-10, 10]）
- `drawdown`：当前净值 / 历史峰值 - 1
